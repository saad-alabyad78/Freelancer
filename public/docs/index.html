Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Headers

Example: multipart/form-data

Accept      

Example: application/json

URL Parameters

industry_name   string   

Example: ipsam

Body Parameters

name   string   

name must be unique company name . Must be at least 3 characters. Must not be greater than 20 characters. Example: kccf

description   string   

Must not be greater than 4000 characters. Example: Nihil sed voluptates delectus et fugiat ducimus.

size   string   

Must be at least 5 characters. Must not be greater than 20 characters. Example: nbdqnszuffucrhwx

city   string   

this must be a syrian city. Example: ÿØŸÖÿ¥ŸÇ

region   string   

Must be at least 3 characters. Must not be greater than 20 characters. Example: rwlvghdsgqdrxm

street_address   string   

Must be at least 3 characters. Must not be greater than 30 characters. Example: r

gallery_images   file[]  optional  

Must be an image. Must be at least 10 kilobytes. Must not be greater than 2000 kilobytes.

contact_links   string[]  optional  

Must be at least 5 characters.

company_phones   string[]  optional  

Must match the regex /^09[0-9]{8}$/.
